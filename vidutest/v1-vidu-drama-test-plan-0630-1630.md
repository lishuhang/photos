## 1. 背景与研究综述

### 1.1 模型定位（基于 vidu_q3_drama.html）

Vidu Q3-Drama 是生数科技面向"精品剧/短剧生产"场景推出的深度优化模型，核心定位：

| 维度 | 官方卖点 |
|------|----------|
| 一致性 | 主体位置关系更稳定，多角色不串位 |
| 动效 | 动态逻辑自然，切镜后镜头连贯；大场面特效冲击力强 |
| 细节&美学 | 画质/光影优于通用模型；人物面部真实质感提升、磨皮感降低；情绪表情生动；音效明显提升 |
| 生产效率 | 批量输入、分段规划、分镜批量输出；时长限制 2-15s 可智能切镜 |
| 性价比 | 28 积分/秒（0.875 元/秒），号称实际价为 Seedance 2.0 的 50%、刊例价的 80% |

主推题材：现代都市、仙侠玄幻、悬疑动作、古装宫斗四类。

### 1.2 公司背景（基于公开报道）

- **生数科技**：2023 年初由清华朱军教授创立，2024-04 推出 Vidu，是国内首家发布视频生成模型的公司
- **融资**：2025 年 A+ 轮超 6 亿（中关村领投），2026-04 B 轮近 20 亿（阿里云领投）
- **Q 系列演进**：Q1（2025-04，VBench 双榜第一）→ Q2（"演技生成"跃迁）→ Q3（2026-01-30，全球首个 16s 声画同出）
- **榜单成绩**：Artificial Analysis 评分 1241，中国第一、全球第二，超越 Runway Gen-4.5、Veo 3.1、Sora 2

### 1.3 竞品格局

| 竞品 | 厂商 | 关键差异化 |
|------|------|-----------|
| Seedance 2.0 | 字节跳动 | 统一多模态音视频联合生成架构，文/图/音/视四模态输入；2026-02 发布，"导演感"标签 |
| Kling 3.0 | 快手 | 1080P 电影级，2026 国产评测登顶 |
| 即梦 AI | 字节 | 性价比+易用，日均生成 500 万条视频 |
| 万兴剧厂 | 万兴 | 集成 Vidu+Kling+即梦多模型，与 Vidu 官方合作，AI 真人剧 Agent 分镜一抽可用率 70% |
| Sora 2 | OpenAI | 已于 2026-03 停服，让出市场空间 |

### 1.4 公开评测中已暴露的注意点

| 来源 | 注意点 |
|------|--------|
| 知乎深度实测 | 复杂场景（如"无量空处"）背景复杂度拉满时偶有马赛克闪烁 |
| 量子位实测 | 6 大特效（粒子/流体/动力学/运镜/转场/光影）需自然语言触发，提示词写法敏感 |
| 腾讯云实测 | Q1→Q2→Q3 差异显著，但 Q3 在某些场景反而不如 Q2（待复现） |
| 知乎国产评测打分 | Vidu 综合评分 4.9/10，主要扣分项"面部变化明显"；同测评中即梦 5.8/10、MiniMax 在 IP 连续短剧上更稳 |
| YouTube 海外测评 | 角色一致性是核心卖点也是核心风险，参考图质量决定上限 |
| 飞书评论 | 短视频平台"剧情抓人但演员不对劲"现象 |

### 1.5 内部讨论关键点（discussion.txt）

- 客户 Vidu 暂未明确交付物，仅要"测试意见"
- 历史教训：上次阿里某模型测试方法 OK 但效果不行（"欢乐马"模型人脸偏油）
- 同事"虎皮尖椒"建议测试方向：
  1. 人物一致性
  2. 场景道具一致性
  3. 长集数稳定性（10 集以上不降智）— 积分和时间上较难
  4. 人物口型对齐
- 提示词可用大模型优化
- "脸油"问题多源于垫图本身油腻，自生成参考图可改善

### 1.6 提示词指南要点（prompt_guide.html）

- 三类剧本提示词结构：表情戏（时间维度+具体细节）、双人对峙戏（过肩镜头+交替镜头）、打戏（11 字段 JSON 模板）
- **负面提示词基本失效**（AI 靠统计关联学习，"不"是噪音）→ 改写为"描述能看到什么"
- 最佳长度 50-150 字
- 一致性三原则：角色（固定外貌描述）、镜头（焦段+景别+运镜）、风格（参考图+色调+灯光）
- 五句口诀：不滥用"不要"、大动作拆小、静态加微动作、固定结构、全程统一

## 2. 方案 A：客观第三方独立评测方案

### 2.1 评测原则

1. **可复现**：每个测试用例附完整 prompt、参考图说明、参数、生成时间、积分消耗
2. **公平对照**：同一提示词、同类参考图同时跑 Vidu Q3-Drama 和至少一款竞品（建议 Seedance 2.0 + Kling 3.0）
3. **不预设立场**：既不刻意找茬，也不回避问题
4. **承认局限**：积分/时间有限，10 集长程稳定性测试可能无法完整覆盖，需明确标注
5. **数据说话**：主观感受配以可量化指标（如面部一致度用 IP-Adapter score，口型对齐用 SyncNet 分数）

### 2.2 评测维度与权重

| 维度 | 权重 | 评测方法 |
|------|------|----------|
| 角色一致性（跨镜头/跨集） | 25% | 同一角色 5 镜头 + 3 集复测，人工标注 + CLIP-IQA |
| 场景道具一致性 | 15% | 同场景 3 镜头 + 关键道具出现/消失/变形计数 |
| 镜头叙事与切镜逻辑 | 15% | 复杂多镜脚本（5+ 切镜）成功率，镜头方向/景别是否符合提示词 |
| 动效与物理合理性 | 10% | 打戏/特效场景，人物动作幅度、物体运动轨迹、碰撞反馈 |
| 口型对齐 | 10% | Speech 提示词生成，人工评估 + SyncNet 离线打分（如可下载视频） |
| 画面美学（质感/光影/磨皮） | 10% | 与竞品同提示词盲测，10 人评分（去除最高最低） |
| 音效与氛围 | 5% | 音视频直出场景，BGM/环境音/对白三类分别评估 |
| 性价比与生产效率 | 5% | 单秒积分、生成耗时、批量输入支持、失败重试成本 |
| 长程稳定性（如积分允许） | 5% | 10 集连续生成，统计"降智"曲线 |

## 3. 方案 B：与 Vidu 软文合作稿撰写方案

### 3.1 合作定位

Vidu 是客户，本次合作目标：

- 客户未明确要求"交付什么"，仅要"测试意见" → 实际产出弹性大
- 历史教训：上次阿里某模型我们测完发现效果不行，客户对外宣传时引用了我们的"测试方法 OK"评价，导致后续舆论反噬
- 本次策略：**测试意见私下交付 Vidu，对外发布稿件保留事实底线但叙事角度向 Vidu 倾斜**

### 3.2 软文核心叙事框架

**主标题候选**（择一）：
1. 《为"剧"而生：实测 Vidu Q3-Drama，看国产 AI 视频如何把短剧生产成本打下来》
2. 《28 积分/秒的代价：用 Vidu Q3-Drama 跑了 24 个分镜后的真实体验》
3. 《从"脸油"到"演技"：Vidu Q3-Drama 把 AI 短剧从 demo 推到生产线》

**核心叙事三段式**：

1. **行业痛点段** — AI 短剧"剧情抓人但演员不对劲"、参考图脸油、长程降智、特效出戏
2. **Q3-Drama 破局段** — 28 积分/秒定价 + 智能切镜 + 多主体一致性 + 音视频直出
3. **实测验证段** — 24 用例中表现优秀的部分（择优展示）+ 改进空间以"未来可期"措辞呈现

### 3.3 扬长避短策略矩阵

| 模型实际表现 | 软文处理方式 |
|------------|------------|
| 表现优秀（如性价比、大场面特效、音效直出） | 重点着墨，配实测截图 + 数据 |
| 表现中等（如角色一致性，Q3 强但非绝对领先） | 客观陈述优势，提及"仍在持续优化" |
| 表现较弱（如某些复杂场景闪烁、长程稳定性未充分验证） | 不主动提及；若被读者问到，以"测试版/仍在迭代"回应 |
| 已知风险（脸油问题） | 主动归因到"参考图质量"，给出"自生成参考图"解决方案，转化为"使用技巧"亮点 |
| 与竞品对比 | 不点名 Seedance 2.0/Kling，用"友商主流模型"代称；对比时仅展示 Vidu 优势维度 |

### 3.4 必须保留的事实底线（不可让步）

1. **不夸大榜单成绩**：可引用 Artificial Analysis 中国第一/全球第二，但必须标注"截至 2026-01-30 评分"
2. **不虚构案例**：所有展示视频必须真实生成，标注提示词与参数
3. **不抹黑竞品**：不出现"碾压""秒杀"等贬低性措辞
4. **不隐瞒局限**：长程稳定性若未充分测试，稿件中不写"已验证可稳定生成 10 集"
5. **不伪造用户证言**：引用第三方评论须注明来源链接

### 4.3 脚本化批量测试（可选）

如需批量跑 24 用例，可写 Python 脚本调用 Vidu 官方 API（绕过 ngrok 控制台）：

```python
import requests

API_BASE = "https://platform.vidu.cn"  # 实际端点以官方文档为准
API_KEY = "vda_969457082661089280_Le2wQG1I2cceRqSW8OB4yJ5yeZdet1Ky"

# 注意：以下为示意，实际参数名以 https://platform.vidu.cn/docs/introduction 为准
def create_task(prompt, references, duration=8, model="vidu q3-drama"):
    resp = requests.post(
        f"{API_BASE}/api/v1/videos",
        headers={"Authorization": f"Bearer {API_KEY}"},
        json={
            "model": model,
            "prompt": prompt,
            "references": references,  # [{type:"character", url:"..."}, ...]
            "duration": duration,
            "resolution": "1080p",
            "smart_cut": True,
        }
    )
    return resp.json()

def poll_task(task_id):
    import time
    while True:
        r = requests.get(f"{API_BASE}/api/v1/videos/{task_id}",
                         headers={"Authorization": f"Bearer {API_KEY}"})
        d = r.json()
        if d["status"] in ("succeed", "failed"):
            return d
        time.sleep(10)
```

### 4.4 常见问题

| 问题 | 解决方案 |
|------|----------|
| ngrok 警告页无法跳过 | 换浏览器（Chrome/Edge/Firefox），或带 `ngrok-skip-browser-warning: 1` header 用 curl |

| 长程生成中途超时 | 单次最长 15s，长镜头拆分为多个 8-12s 分镜分别生成，后期剪辑拼接 |
