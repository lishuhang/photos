## 一、画布是什么

这个画布是一个 **基于 React Flow 的节点式（node-based）AI 影像创作工具**，类似 Houdini / ComfyUI / n8n 的工作方式——把"风格""场景""角色参考图""文本提示词""生成的图片""生成的视频""故事板""剪辑"等都做成一个个**节点**，然后用**连线（Edge）**把它们串成一条"从设定到成片"的工作流。

核心思想：**每个节点是独立资产，连线代表数据/引用流向，整张画布就是一部片子的完整生产图谱**。

## 三、节点类型全景

通过分析 3 个赛事参考案例，画布共有 **8 种节点类型**：

| 节点类型 | DOM class | 颜色/形状 | 作用 |
|---|---|---|---|
| **风格（style）** | `react-flow__node-style` | 见截图 | 定义画风提示词，可开关"仅使用风格提示词" |
| **场景（scene）** | `react-flow__node-scene` | 见截图 | 定义场景环境提示词，可添加素材 |
| **文本（text）** | `react-flow__node-text` | 见截图 | 纯文本节点，写提示词或描述词 |
| **图片（image）** | `react-flow__node-image` | 见截图 | 图片资产——可上传、可由提示词生成、可从视频截帧 |
| **视频（video）** | `react-flow__node-video` | 见截图 | 视频资产——可由图片+提示词生成，含时长信息 |
| **故事板（storyboard）** | `react-flow__node-storyboard` | 见截图 | **核心编排节点**——把分镜号+prompt+参考图+视频时长编排在一起 |
| **视频编辑/剪辑（video_editor）** | `react-flow__node-video_editor` | 见截图 | 剪辑台——把多个视频按顺序拼成成片，可下载 |
| **编组（group）** | `react-flow__node-group` | 见截图 | 视觉分组，把相关节点圈在一起，不参与数据流 |

### 3.1 节点结构

每个节点（除 group 外）有统一结构：

```
┌─────────────────────────┐
│  输入连接点（左侧）        │ ← 接收上游数据/引用
├─────────────────────────┤
│  节点标题（如"风格·手绘国漫风"）│
│  @ 节点名                │
│                         │
│  [内容区——提示词/图片/视频] │
│                         │
│  [可选] 仅使用风格提示词（开关）│
│  [可选] 添加素材（按钮）     │
├─────────────────────────┤
│  输出连接点（右侧）        │ → 输出给下游
└─────────────────────────┘
```

## 四、连线规则（Edge）

连线代表**引用关系**——上游节点的输出作为下游节点的输入。从案例 1 的 62 条边分析，主要连线模式有：

### 4.1 七种典型连线模式

| 模式 | 上游 → 下游 | 含义 |
|---|---|---|
| **风格→图片/视频** | style → image/video | 把画风提示词注入生成 |
| **场景→图片/视频** | scene → image/video | 把场景环境注入生成 |
| **文本→图片** | text → image | 用文本提示词生图 |
| **图片→图片** | image → image | 图片编辑/再生成（如三视图→角色特写） |
| **图片→视频** | image → video | 图生视频 |
| **视频→图片** | video → image | 视频截帧 |
| **视频→剪辑** | video → video_editor | 把视频加入剪辑时间线 |

### 4.2 案例 1 的实际连线示例（前 10 条）

```
裁剪.png           → 图片5-1          （裁剪图作为源图生成新图）
裁剪.png           → 图片10-2
文本1（女孩多手描述）→ 图片10-2        （文本提示词生图）
图片10-2           → 视频1（00:08）    （图生视频）
图片1-3            → 视频1            （多张参考图同时输入一个视频）
image.png          → 图片3            （参考图生图）
396B...F041.png    → 图片3            （第二张参考图）
图片3-4            → 视频1（00:05）    （图生视频）
文本1              → 视频1            （文本也直接进视频）
图片1              → 视频8-1          （图生视频）
```

### 4.3 连线操作

- **画线**：从上游节点的"输出连接点"（右侧）拖到下游节点的"输入连接点"（左侧）
- **删除线**：点选连线后按 Delete
- 一对多、多对一都支持（如多张参考图同时连到一个视频节点）

## 五、故事板节点——核心编排利器

故事板（storyboard）是案例 1 中最强大的节点，它把**一整支片子的分镜表+prompt+参考图引用+视频时长**全部编排在一个节点里。

### 5.1 故事板节点结构

```
故事板1
├── 镜头号1
│   ├── [本镜提示词] 将女孩的裤子改成深棕色，衬衣白色...
│   ├── 视频1  00:08  00:00 / 00:08
│   │   └── [本镜分镜描述] 0-3S：水平构图，女孩在街上走...
│   │       └── @参考图3，脸上透出焦虑和劳累。
├── 镜头号2
│   ├── [本镜提示词] 去掉女孩包上的纸...
│   ├── 视频2  00:10  00:00 / 00:10
│   │   └── [本镜分镜描述] 0-3S：水平构图，女孩@参考图1在街上走...
│   │       └── 3-5S：进入旋转门@参考图3...
│   │       └── 5-8S：整个人变得非常疲惫@参考图2...
├── 镜头号3
│   └── ...
├── 镜头号4
│   ├── 视频4  00:10
│   │   └── 00:00-02:00  中景  侧平角  [详细动作描述]  运动强度：中(5)
│   │   └── 02:00-03:50  特写  俯冲角度  [详细动作描述]  运动强度：极高(10)
│   │   └── 03:50-05:50  大特写  微距视角  [详细动作描述]  运动强度：低(2)
│   │   └── ...（5段分镜表，含景别/角度/动作/运动强度）
├── 镜头号5
│   └── ...
└── 镜头号6
    └── ...
```

### 5.2 故事板 prompt 的三种写法

**写法 A：简洁改造式（镜头 1-3）**

```
镜头号1
将女孩的裤子改成深棕色，衬衣白色，浅灰绿色马甲。角色表情开心，头发柔顺，画风严格按照参考图片。
左侧1/3 区域为超大高清面部特写...右侧 2/3 区域整齐排布角色三张全身三视图...
```

适用于：对参考图做局部改造、生成三视图。

**写法 B：时间轴分镜式（镜头 4）**

```
镜头号4
00:00-02:00  中景  侧平角  办公室全黑白。女孩坐在工位前，背影显得紧绷...  运动强度：中(5)
02:00-03:50  特写  俯冲角度  快切镜头：特写多双纤细的手指在机械键盘上飞速敲击...  运动强度：极高(10)
03:50-05:50  大特写  微距视角  女孩苍白饱满的额头，一颗晶莹剔透的汗珠...  运动强度：低(2)
```

适用于：单镜头多段动作，每段标明时间、景别、角度、动作描述、运动强度。

**写法 C：参考图引用式（镜头 2、5）**

```
镜头号2
0-3S：水平构图，女孩@参考图1在街上走，走到公司门口进去
3-5S：进入公司的旋转入门后@参考图3，周围的颜色慢慢褪去，变为黑白色调。
5-8S：整个人也变得非常疲惫@参考图2，脸上透出焦虑和劳累。

镜头号5
将图2@参考图2闹钟放在图1@参考图1桌子上，并给在桌子上的闹钟一个特写镜头。
要求透视保持一致
```

适用于：在描述中嵌入 `@参考图N` 引用，模型会自动把对应连线上的参考图喂进去。

### 5.3 故事板与外部节点的连线

故事板节点本身可以接收**多个图片节点**作为输入（通过输入连接点），这些图片就成了故事板内的"@参考图1""@参考图2"……按连入顺序编号。

## 六、sc 分组模式（案例 2/3 的工作流）

案例 2 和 3 用了另一种组织方式——**按场景分组的"sc 工作流"**：

### 6.1 结构

```
sc01（编组）
├── sc01-图片1（image）        ← 参考图/生成图
├── sc01-图片1-描述词（text）   ← 该图的 prompt
├── sc01-视频（video）          ← 由图片生成的视频
└── sc01-视频 → 剪辑预览1（video_editor）

sc02（编组）
├── sc02-图片1
├── sc02-图片2
├── sc02-视频
└── sc02-视频 → 剪辑预览1

...（sc03 ~ sc07 同理）
```

### 6.2 "描述词"文本节点的 prompt 风格

案例 2 的 sc01-图片1-描述词（完整）：

```
尔特写镜头，斜角对角线构图，左上部大面积展现粗糙、有颗粒感的手工米白纸张，
右下角是复古打字机精密的金属导轨与打字杆。金属字模正精准撞击在纸张中央，
清晰印出"happy horse"字样。在这一行字的最左边，同样用印有...
```

特点：**镜头语言 + 构图 + 物体细节 + 文字内容**全部写进 prompt，一个文本节点对应一张图。

### 6.3 两种模式对比

| 维度 | 故事板模式（案例1） | sc 分组模式（案例2/3） |
|---|---|---|
| 组织方式 | 单一大节点装所有镜头 | 每个场景独立编组 |
| prompt 位置 | 全部写在故事板节点里 | 每张图配一个独立"描述词"文本节点 |
| 参考图引用 | @参考图N 内嵌引用 | 直接连线 text→image |
| 适合场景 | 镜头数多、关系复杂 | 场景独立、每镜独立调优 |
| 改 prompt | 改故事板一个节点 | 改对应 text 节点 |
| 看全局 | 故事板里看全部分镜 | 画布上看 sc 分组布局 |

## 七、视频生成与剪辑

### 7.1 视频节点

每个视频节点显示：
```
视频1
00:08          ← 总时长
00:00 / 00:08  ← 当前播放位置 / 总时长
```

视频由**图片节点 + 提示词**连线生成。可连多张图（多参考图）。视频时长常见 5s/8s/10s。

### 7.2 视频截帧

视频可以反过来生成图片节点——"截帧 00:00:04"就是从视频第 4 秒截一张图，用于：
- 作为下一镜的首帧参考
- 作为再生成的输入

### 7.3 剪辑节点（video_editor）

```
剪辑预览1-副本1
剪辑工具  00:00:00:00 / 00:00:38:11
视频1  00:05
视频2  00:05
视频3  00:04
视频4  00:03
视频5  00:02
...
[添加视频]  [下载]
```

- 把多个视频节点按顺序拼成成片
- 显示每个视频时长 + 总时长
- 可下载成片

## 九、给《口技》测试片的画布搭建建议

### 9.1 建议采用 sc 分组模式

理由：
- 《口技》14 镜跨 4 个场景（厅事/卧房/街市/火警），sc 分组模式按场景组织最清晰
- 每镜可独立调 prompt，互不干扰
- 与三视图素材包的"每人物3视图"结构天然契合

### 9.2 建议的画布结构

```

左上区：设定层（已有3节点，需补全）
├── 风格·手绘国漫风 ✓ 已有

├── 场景·长安夜街场景 ✓ 已有
├── 场景·厅事（待建）
├── 场景·卧房（待建）
├── 场景·街市火警（待建）
├── 图片·口技者正面（待导入三视图）
├── 图片·口技者侧面（待导入）
├── 图片·口技者背面（待导入）
├── 图片·妻正面/侧面/背面（待导入）
├── 图片·夫正面/侧面/背面（待导入）
├── 图片·大儿正面/侧面/背面（待导入）
├── 图片·婴儿正面/侧面/背面（待导入）
```

### 9.3 每个 sc 分组内的标准结构

```
scXX（编组）
├── 文本·scXX-描述词  ←  写本镜的完整 prompt（含画风+场景+人物+动作+镜头语言）
│   ↑ 连线：风格节点 + 场景节点 + 人物参考图节点
├── 图片·scXX-生成图  ←  由文本节点生成
├── 视频·scXX-视频    ←  由图片节点生成（图生视频）
└── 视频·scXX-视频 → 剪辑节点
```

### 9.4 V01 的 prompt 模板示例

文本节点 `sc01-描述词` 内容：

```
90年代中国手绘动画风格，参考《大头儿子小头爸爸》《海尔兄弟》线条质感。
清朝中期长安夜街远景缓推至厅事中景：街市三位宾客挑灯沿青石板路赴宴，
衣袍随夜风轻摆；切至厅事内仆人双手扶起八尺绘山水屏风，屏风前木桌上
摆折扇与抚尺；末帧一位45岁清代男性口技艺人拱手入座屏障中。
平涂色块，深褐色手绘勾线，整体暖黄烛光。
禁止日漫笔触、赛璐璐渐变、真人实拍、3D渲染、大眼睛尖下巴。
16:9横构图。
```

连线：
- `风格·手绘国漫风` → `文本·sc01-描述词`
- `场景·长安夜街场景` → `文本·sc01-描述词`
- `图片·口技者正面` → `文本·sc01-描述词`（作为人物锚点）
- `图片·道具组` → `文本·sc01-描述词`（作为道具锚点）
- `文本·sc01-描述词` → `图片·sc01-生成图`（生成首帧图）
- `图片·sc01-生成图` → `视频·sc01-视频`（图生视频）

## 十一、注意事项

1. **"使用"案例会替换当前画布**——务必先建空项目再使用案例
2. **风格节点的"仅使用风格提示词"开关**——开启时该风格只作为画风注入，不生成图；关闭时会用风格提示词生成图
3. **故事板的 @参考图N**——按图片连入故事板的顺序编号，注意连线顺序
4. **视频时长**——常见 5s/8s/10s，生成时选好；剪辑节点会按视频顺序拼接
5. **截帧节点**——可从视频任意时刻截图，常用于"上一镜末帧→下一镜首帧"的过渡
6. **编组（group）**——只是视觉框，不参与数据流，用于组织相关节点
