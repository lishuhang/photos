
# HappyHorse 1.1分镜测试提案v1

> 以中学语文课文《口技》为内核，串联五大升级 × 五大测试要点的连贯故事化测试方案

分镜数量：15镜（5幕），预计总时长约80秒

## 测试总体需求

用户公认HappyHorse 1.0的强项：

- 价格低（0.44元/秒起）
- 速度快（H100上5秒1080P视频约38秒）
- 镜头感
- 音视频联合生成

1.0短板：

1. “油光感”“过度锐化”等
2. 风格漂移：“图生视频阶段模型跑着跑着就变成真人风格”
3. 多镜头调度锚点漂移
4. 角色一致性有时保持不住
5. 多人对话、复杂音乐和精细拟音仍需后期

HappyHorse 1.1相比1.0“五大维度系统性升级”：

1. 动态表现力：针对1.0版本部分画面动作迟缓、节奏不足的问题，优化了运动建模与时序一致性能力，提升了动作的连贯性和力量感。
2. 主体一致性：支持9张角色参考图同时输入，保持商品细节、品牌元素、角色与场景的灵活组合和稳定性，多分镜与N宫格参考理解增强。多角色场景中角色“变脸”情况可控。
3. 指令遵循：对于打斗等高强度动态场景，简洁的提示词便可理解。对于复杂提示词，镜头编排稳定性增强，可高质量完成多场景、多角色的连贯演绎。
4. 视觉质感：此前用户反馈集中的“油光感”“过度锐化”等问题得到显著缓解，保留了痘印、法令纹、毛孔等细节的真实感，
5. 音频能力：台词表达更加自然，语速、停顿、语气能够根据场景与情绪变化动态调整。用户亦可在提示词中描述背景、环境音效等。

测试要点：

1. 拟真人镜头去油光感：近景细节，如正反打（两人对话时，镜头越肩拍摄对方）
2. 9张参考图
3. 少提示词的智能叙事编排 + 复杂提示词编排，形容同一场景
4. 多角色一致性测试
5. 检查上述内容的配音，同时做一个专门体现声音的场景

附加要求：

希望整个内容处于一个连贯的故事情节中，自然融入所有的功能点，扬长补短（短板可由官方提供，或在测试过程中观察到时加以规避）

如执行时间充裕，也可考虑同步换用1.0版本抽一个对照组。

## 为什么选《口技》

该文本对国内观众有亲切感，是熟悉的语料；其中的部分场景可以映射本次测试的几种能力。

1. 拟真人镜头：静态的妇人哄孩子等
2. 多参考图 + 多角色一致性测试：妇人、丈夫、婴儿、犬等角色固定；抚尺、扇、桌、椅等道具，家居场景固定
3. 提示词遵从：结尾 百千齐作、宾客离席等可用少提示词和复杂提示词分别测试
4. 配音：在镜头中加入男声朗读原文混音，测试是否有效
